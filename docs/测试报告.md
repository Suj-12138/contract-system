## 三、覆盖分析

| 覆盖维度 | 说明 |
|----------|------|
| **正常路径** | 每个 API 端点至少 1 条成功路径测试 |
| **异常路径** | 缺字段、参数非法、资源不存在 |
| **认证边界** | 未登录 (401)、已登录正常访问 |
| **权限边界** | admin/handler/approver 角色隔离，越权返回 403 |
| **业务规则** | 合同状态流转合法性、两级审批顺序、越级阻止 |
| **数据完整性** | 原子写入、并发读、异常回滚 |

## 四、执行方式

```bash
cd backend
pip install -e ".[dev]"
pytest -v
```

测试基于内存中的 httpx AsyncClient，无需启动服务器。每个测试用例通过 `tmp_path` 使用独立临时文件，互不干扰。

## 五、已知局限

1. **到期提醒扫描**未覆盖自动化测试——该功能依赖定时任务，适合手工验证
2. **前端页面交互**未做 Playwright 自动化——当前为 API 层测试 + 手工 UI 走查
3. **并发写入竞争**未做压力测试——存储层有线程锁，但未验证极端并发场景
