# 企业合同管理系统 — 需求分析文档

> 版本：v1.0 | 日期：2026-07-08 | 团队：5 人

---

## 一、项目背景

企业内部合同管理目前依赖纸质或 Excel 方式，存在审批流程不透明、合同到期无人跟进、模板复用率低等问题。本项目旨在开发一个企业合同全生命周期管理系统，实现合同的在线拟制、审批、签订、履行跟踪和归档。

纯内部系统，仅企业员工使用。

---

## 二、用户角色

| 角色 | 标识 | 职责概述 |
|------|------|----------|
| 经办人 | `handler` | 创建、编辑、提交、签订、归档自己的合同 |
| 审批人 | `approver` | 审批待处理的合同（通过/驳回），查看审批记录 |
| 管理员 | `admin` | 创建用户、管理模板、全局查看合同、查看数据看板 |

## 三、功能需求

### FR1：用户认证

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR1.1 | 用户使用用户名或邮箱+密码登录 | 高 |
| FR1.2 | 登录后保持 8 小时会话 | 高 |
| FR1.3 | 管理员创建新用户账号 | 高 |
| FR1.4 | 管理员启用/禁用用户，禁用后自动踢出 | 中 |
| FR1.5 | 首次启动自动创建管理员（通过环境变量配置） | 高 |

### FR2：合同管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR2.1 | 经办人创建合同（输入标题、对方名称、金额、到期日期） | 高 |
| FR2.2 | 经办人编辑草稿或被驳回的合同 | 高 |
| FR2.3 | 经手人查看自己的合同列表，支持按状态筛选 | 高 |
| FR2.4 | 管理员查看全部合同列表 | 中 |
| FR2.5 | 合同详情包含内容、审批进度、操作历史 | 高 |
| FR2.6 | 按合同标题和对方名称关键词搜索 | 中 |

### FR3：合同模板

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR3.1 | 管理员创建合同模板，定义可填字段（文本、数字、日期、文本域） | 高 |
| FR3.2 | 管理员编辑和启用/停用模板 | 中 |
| FR3.3 | 经办人创建合同时选择模板，根据模板字段动态生成表单 | 高 |
| FR3.4 | 经办人也可不以模板为基础自由填写合同 | 中 |

### FR4：审批流转

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR4.1 | 固定两级审批：一级（部门主管）→ 二级（法务） | 高 |
| FR4.2 | 一级审批通过后，二级才能审批 | 高 |
| FR4.3 | 任一级驳回，合同回到草稿状态，经办人可修改后重新提交 | 高 |
| FR4.4 | 两级均通过后，合同进入已通过状态 | 高 |
| FR4.5 | 审批时需录入审批意见 | 低 |
| FR4.6 | 经办人创建合同时指定两级审批人 | 高 |

### FR5：合同生命周期

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR5.1 | 合同状态线性流转：草稿 → 审批中 → 已通过/已驳回 → 已签订 → 履行中 → 已到期/已续签 → 已归档 | 高 |
| FR5.2 | 经办人确认签订 | 高 |
| FR5.3 | 经办人或管理员归档已到期/已续签合同 | 中 |

### FR6：到期提醒

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR6.1 | 系统自动扫描即将到期合同（默认 30 天内） | 中 |
| FR6.2 | 向合同经办人发送站内消息提醒 | 中 |
| FR6.3 | 可通过环境变量配置提前提醒天数 | 低 |

### FR7：站内消息

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR7.1 | 查看收件箱，区分未读/已读 | 高 |
| FR7.2 | 点击消息标记已读 | 中 |
| FR7.3 | 导航栏显示未读消息数 | 中 |

### FR8：管理员功能

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR8.1 | 用户列表查看 | 中 |
| FR8.2 | 创建用户（设定用户名、邮箱、密码、角色） | 中 |
| FR8.3 | 启用/禁用用户 | 中 |
| FR8.4 | 数据看板：合同总数、状态分布、审批通过率、即将到期数 | 中 |

### FR9：操作审计

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR9.1 | 记录每次合同操作（创建、提交、审批、签订、归档） | 低 |
| FR9.2 | 合同详情页展示操作历史 | 低 |

## 四、非功能需求

| 编号 | 类型 | 需求 |
|------|------|------|
| NFR1 | 性能 | 支持至少 5 人并发操作（单进程足够） |
| NFR2 | 数据安全 | 密码 Argon2id 哈希存储，不允许明文 |
| NFR3 | 认证安全 | Session Cookie: HttpOnly, SameSite=Lax，防 XSS/CSRF |
| NFR4 | 数据一致性 | JSON 文件原子写入（临时文件 + os.replace），不会读到半截数据 |
| NFR5 | 并发安全 | 写操作使用 threading.RLock 加锁 |
| NFR6 | 可用性 | 错误信息用中文，操作失败有明确提示 |
| NFR7 | 兼容性 | 支持 Chrome / Edge 现代浏览器 |
| NFR8 | 部署 | 无需外部数据库，解压即用，Python 3.10+ 即可启动 |

## 五、业务规则

| 编号 | 规则 |
|------|------|
| BR1 | 合同只能由经办人编辑，且仅限草稿或已驳回状态 |
| BR2 | 驳回的合同回到草稿状态，其它状态均不可回退 |
| BR3 | 审批记录保留不删除，驳回后重新提交仅重置决策状态 |
| BR4 | 用户被禁用时，该用户所有活跃会话自动失效 |
| BR5 | 两级审批必须按顺序进行，不可越级 |
| BR6 | 合同提交审批时创建两条审批记录，重新提交时重置 |
| BR7 | 签订后合同自动进入履行中状态 |

## 六、用例描述

### UC1：完整审批流程（主成功场景）

| 步骤 | 角色 | 操作 |
|------|------|------|
| 1 | 管理员 | 登录系统 |
| 2 | 管理员 | 创建合同模板"采购合同"（含甲方、乙方、金额、交付日期、条款 5 个字段） |
| 3 | 管理员 | 创建经办人账号 handler1 |
| 4 | 管理员 | 创建审批人账号 approver1（主管）、approver2（法务） |
| 5 | 经办人 | 登录系统 |
| 6 | 经办人 | 选择"采购合同"模板创建合同，填写各字段，指定审批人 |
| 7 | 经办人 | 提交审批 |
| 8 | 审批人1（主管） | 登录，看到待审批合同，点击通过 |
| 9 | 审批人2（法务） | 登录，看到待审批合同，点击通过 |
| 10 | 经办人 | 登录，合同已通过，点击确认签订 |
| 11 | 管理员 | 登录，数据看板显示合同数+1 |

### UC2：审批驳回与重提（替代场景）

| 步骤 | 角色 | 操作 |
|------|------|------|
| 1-7 | 同 UC1 | 经办人创建合同并提交审批 |
| 8 | 审批人1（主管） | 审批驳回，填写意见"交付日期不合理" |
| 9 | 经办人 | 收到驳回消息，编辑合同修改交付日期 |
| 10 | 经办人 | 重新提交审批 |
| 11 | 审批人1 | 审批通过 |
| 12 | 审批人2 | 审批通过 |
| 13 | 经办人 | 确认签订 |

### UC3：到期提醒

| 步骤 | 角色 | 操作 |
|------|------|------|
| 1 | 经办人 | 签订了一份 3 个月后到期的合同 |
| 2 | 系统 | 在到期前 30 天自动扫描，向经办人发送站内消息"合同即将到期" |
| 3 | 经办人 | 登录，收到提醒，跟进处理 |

## 七、需求范围边界

### MVP 内

- 用户注册/登录/退出（Cookie Session）
- 合同从草稿到归档的完整生命周期管理
- 固定两级审批且按顺序流转
- 合同模板创建与使用
- 标题和对方名称的简单全文检索
- 基于到期日期的站内消息提醒
- 管理员用户管理和数据看板
- 合同操作审计日志

### MVP 外（本期不实现）

- 附件上传与预览
- 邮件/短信外部通知
- 第三方电子签名集成
- SLA 超时告警
- 高级搜索（多条件筛选、日期范围）
- 数据导出（Excel/PDF）
- 批量操作
- 合同版本对比
- 密码找回与验证码
- 审批链自定义配置（动态审批层级）
- 定时任务与后台调度
