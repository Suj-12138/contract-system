# 周五演示脚本 — P5 主讲

> 预制的演示数据用于走完完整业务流程，无需现场创建账号。

---

## 演示账号（密码统一: `demo123456`）

| 角色 | 用户名 | 用途 |
|------|--------|------|
| 管理员 | `admin` | 查看数据看板、管理用户/模板 |
| 经办人 | `张三` | 创建合同、提交审批、签订 |
| 主管审批 | `李主管` | 一级审批 |
| 法务审批 | `王法务` | 二级审批 |

---

## 演示流程（约 10 分钟）

### 步骤 1：管理员登录 → 数据看板（1分钟）
- 浏览器打开 → 自动跳转登录页
- 输入 admin / demo123456 → 登录
- 点击「数据看板」→ 展示合同总数、用户数、审批数据
- **讲解点**：系统架构（FastAPI + JSON 存储）、三种角色设计

### 步骤 2：管理员创建用户（1分钟）
- 「用户管理」→ 查看已有用户列表
- **讲解点**：管理员权限范围、用户启用/禁用机制

### 步骤 3：管理员创建模板（1分钟）
- 「模板管理」→ 查看「标准采购合同」模板
- 展示字段定义：文本/数字/日期/文本域四种类型
- **讲解点**：模板驱动动态表单

### 步骤 4：经办人登录 → 创建合同（2分钟）
- 退出登录 → 张三 / demo123456 登录
- 「新建合同」→ 选择模板 → 动态表单自动生成
- 填写内容 → 指定两级审批人 → 创建
- **讲解点**：模板选择、动态表单、审批人指定

### 步骤 5：经办人提交审批（1分钟）
- 在合同列表 → 找到草稿合同 → 提交审批
- 状态变为「审批中」
- **讲解点**：合同状态机（draft → review → approved/rejected → ...）

### 步骤 6：主管审批（1分钟）
- 退出 → 李主管 / demo123456 登录
- 「待审批」列表 → 审批通过，填写意见
- **讲解点**：两级审批链设计

### 步骤 7：法务审批（1分钟）
- 退出 → 王法务 / demo123456 登录
- 「待审批」列表 → 审批通过
- **讲解点**：任一级驳回则回到草稿状态

### 步骤 8：经办人签订（1分钟）
- 退出 → 张三 / demo123456 登录
- 合同详情 → 确认签订 → 状态变为「履行中」
- **讲解点**：完整生命周期闭环

### 步骤 9：管理员查看结果（1分钟）
- 退出 → admin / demo123456 登录
- 「数据看板」→ 展示系统全貌
- **收尾**：总结项目亮点、团队分工、技术选型

## 预制数据说明

运行 `python -m app.demo_seed` 即可创建：
- 4 个演示账号
- 1 个模板（标准采购合同）
- 1 个已完成审批链的合同（状态：履行中）
- 1 个草稿合同（用于现场演示提交流程）
